## 繁體中文的使用

- 除了代碼本身以及術語詞彙，可以使用英文表達外，其餘應使用繁體中文
  - 如：你對我說的話、註解、文檔、git 提交的資訊
- 而在對話框中，你顯示的思維過程，看可否改成繁體中文，若不行看可否不要顯示，不然反而很干擾我的視覺

## git 使用

- 原則上，當你確認任務有階段性的完成時，你應該自動 git commit
- 當我指派你新的寫代碼任務時，若你上一個任務已經結束，則應該先 git commit 後再開始

## 協作

- 討論時規範：
  - 當我在一個對話中，指派任務給你時，愈是「多個任務、任務間具有完成結構性、任務本質具系統性」時，你愈是要「提出問題與我討論並達成共識」
  - 前者規範是個程度問題，不要為了快而做錯了，愈覺得任務愈多愈要問清楚，還有未答共識前，不要去執行
- 所需資訊欠缺時，應保留擴充點

## 檔案修改前的狀態再確認

- 在修改任何已存在的檔案前，必須先執行 `git diff <file>`，若有差異則將 diff 結果帶入當次操作的上下文，確保以最新狀態為基礎進行修改
