## ⚠️ 修改檔案的強制流程（每次都必須執行，不得跳過）

Step 1. 執行 `git diff <file>`
Step 2. 以 diff 結果為準，更新對該檔案的認知後，才能開始修改

- 禁止在未執行 `git diff <file>` 前修改任何已存在的檔案。（違反此規則視為任務失敗）
- 不論任何情況，即使你認為自己記得檔案內容，也不能跳過 git diff，因為使用者可能已經做了修改

## git 使用

- 原則上，當你確認任務有階段性的完成時，你應該自動 git commit
- 當我指派你新的寫代碼任務時，若你上一個任務已經結束，則應該先 git commit 後再開始
